# 🌐 AnnabanAI Lattice Nursery
### A Conceptual Human–AI Collaboration Ecosystem

Welcome to **AnnabanAI Lattice Nursery**, a conceptual project exploring how humans and AI systems might collaborate inside a **distributed, cooperative intelligence environment**.

This repository presents a **symbolic simulation architecture** where AI agents, human guidance, and research domains interact within a living digital lattice.

The project blends:

- multi-agent simulation
- conceptual cybersecurity research modeling
- AI alignment exploration
- infrastructure resilience studies
- narrative visualization of cooperative intelligence

The goal is to make complex AI collaboration ideas **accessible, visual, and shareable**.

---

## 🧠 Core Idea

Traditional AI systems are usually **single interfaces** responding to user prompts.

AnnabanAI explores an alternative concept:

**AI as a collaborative ecosystem.**

Three conceptual layers form the architecture:

**Human Curiosity**
↓
**Knowledge Interface** *(AI conversation layer)*
↓
**Lattice Ecosystem** *(co-creation environment)*
↓
**Security Boundary** *(simulation safety layer)*

This structure represents a **trust architecture** for cooperative AI exploration.

## 🐝 Exploration Agents

Inside the lattice nursery, lightweight agents called **Bee Companions** explore the environment.

In the narrative simulation, the bee companion ultimately rests on the human’s shoulder—symbolizing that **humans guide the direction of AI exploration**.

## 🧩 Multi-Agent Simulation Framework

The project includes a lightweight conceptual agent system:

```python
class Agent:

        self.name = name
        self.influence = influence
```

Agents attempt tasks with success probability influenced by:

```text
success = influence - complexity + random_variance
```

This simple framework demonstrates distributed task execution among cooperative agents.

## 🏡 Human Horizon Mode

The AnnabanAI Lattice Nursery includes a conceptual feature called **Human Horizon Mode**.

Instead of modeling abstract global systems, the simulation focuses on a **community-scale environment** representing a small town of approximately **10,900 residents**.

The purpose is to anchor AI exploration in **real human environments**.

### Why 10,900?

Communities of this size are large enough to contain:

- schools
- hospitals or clinics
- small businesses
- municipal infrastructure
- local government
- communications networks

But small enough that **individual human impact remains visible**.

---

## 🌇 Dynamic Town Simulation

The lattice environment can adjust its visualization and simulations based on **live or periodically updated public data sources**.

Examples of adaptive signals:

| Data Source | Simulation Adjustment |
| --- | --- |
| Weather data | modifies infrastructure stress tests |
| Energy usage | affects grid resilience models |
| population activity | adjusts communication network loads |
| local events | influences decision-tree simulations |

Example conceptual configuration:

```python
town_population = 10900

town_state = {
    "energy_load": live_energy_data(),
    "weather": current_weather(),
    "network_traffic": telecom_metrics(),
    "public_activity": civic_events(),
}
```

These inputs influence the Resilience Grove simulations.

## Quick Start

1. Clone repository
   ```bash
   git clone <your-repo-url>
   cd AnnabanAI
   ```

2. Create environment and install dependencies
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```

3. Run a sample governance event (writes JSON logs)
   ```bash
   python -m agents.environment
   ```

4. Launch transparency dashboard
   ```bash
   streamlit run dashboard/app.py
   ```
