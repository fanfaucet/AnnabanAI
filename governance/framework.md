# AnnabanAI Governance Framework (Prototype)

## Decision Lifecycle
1. Proposal is submitted with normalized rubric fields (`human_benefit`, `safety`, `feasibility`).
2. Council agents evaluate proposal and vote.
3. Consensus score is computed.
4. If a Human Veto Node is required, human steward must approve.
5. Final status is logged with full trace.
6. Value Return Protocol simulation records intended public-benefit routing.

## Human Veto Node Policy
- Trigger for high-impact proposals or any policy-sensitive domain.
- Human steward identity is explicitly stored in logs.
- A veto always supersedes machine consensus.

## Value Return Protocol (VRP)
VRP is represented in this prototype as a conceptual routing layer:
- Community Education Fund
- Open Infrastructure Commons
- Digital Public Health Lab

No real financial transfer occurs in this version.
